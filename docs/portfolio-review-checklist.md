# Portfolio Review Checklist

Use this checklist before sending the repo to an employer or recruiter.

## Hiring-Manager Quality

- [x] README explains the project quickly.
- [x] Repo shows both product thinking and technical thinking.
- [x] Docs are written for business and technical readers.
- [x] Code examples are small enough to review quickly.
- [x] Tests demonstrate basic engineering discipline.
- [x] Security boundary is explicit and professional.

## Public Safety

- [x] No production dashboard code.
- [x] No live URLs, webhook endpoints, or Sheet IDs.
- [x] No API keys, tokens, secrets, or environment values.
- [x] No real client names, emails, or records.
- [x] No Make.com blueprints or exact automation logic.
- [x] No proprietary scoring, prioritization, or recovery formulas.
- [x] No private prompts or advisory logic.

## Interview Positioning

This repo should position the owner as someone who can:

- Understand messy business problems
- Turn business workflows into technical architecture
- Build safe MVPs
- Communicate clearly with stakeholders
- Protect sensitive systems while still showing meaningful work
